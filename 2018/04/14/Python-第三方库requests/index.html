Requests 会自动解码来自服务器的内容, 大多数的 unicode 字符集都能被无缝解码

属性 说明
r.status_code http响应状态码,200表示成功
r.raw 获取来自服务器的原始套接字响应, r.raw.read()获取
r.content 以字节的方式访问请求响应体
r.text 以字符串方式访问请求响应体
r.headers 以字典对象存储服务器响应头
r.encoding 查看Requests编码，且使用此属性来改变它
r.json() 内置的 JSON 解码器，助你处理 JSON 数据
r.raise_for_status() 失败请求(非200响应)抛出异常

text和content的不同之处 :

提取文本就用text
提取图片、文件，就要用到content

错误与异常

遇到网络问题（如：DNS 查询失败、拒绝连接等）时，Requests 会抛出一个 ConnectionError 异常。

如果 HTTP 请求返回了不成功的状态码， Response.raise_for_status() 会抛出一个 HTTPError 异常。

若请求超时，则抛出一个 Timeout 异常。

若请求超过了设定的最大重定向次数，则会抛出一个 TooManyRedirects 异常。

所有Requests显式抛出的异常都继承自 requests.exceptions.RequestException 。
